# Monster YAML Reference

This document provides a comprehensive reference for all YAML keys used in the D&D monster statblock system.

## Key Categories

- **(USER)**: Keys that users should include in their YAML monster files
- **(CALCULATED)**: Keys automatically generated by `angrycalc` (may be overridden)
- **(OPTIONAL)**: Keys that may be present but are not required

## Workflow

Monsters can be created in two ways:

1. **From Angry Monster Creator JSON**: Use `monster-json-to-yaml` to convert JSON files to YAML
2. **Direct YAML authoring**: Create YAML files following this reference

After creating a monster YAML, run it through `angrycalc` to calculate derived values like CR, XP, and text renderings.

## Required Fields

The following fields should typically be present for a functional monster:

- `NAME`: Monster's name
- `TIER`: Power tier (apprentice, journeyman, adventurer, veteran, champion, legendary)
- `ORGANIZATION`: Encounter organization (solo, pair, group, gang, mob, party)
- `SIZE`: Creature size (tiny, small, medium, large, huge, gargantuan)
- `MONSTER TYPE`: Type of creature (e.g., "Humanoid", "Beast", "Ooze")
- Ability scores: `STR`, `DEX`, `CON`, `INT`, `WIS`, `CHA`
- `MAX HP`: Maximum hit points
- At least one of `DPR`, `SAVE DC`, or `ATTACK BONUS`

## Calculated Fields and the OVERRIDES Key

`angrycalc` automatically calculates many fields. If you want to override a calculated value, add the field name to the `OVERRIDES` list:

```yaml
OVERRIDES:
  - AC
  - DPR
AC: 15  # Will not be recalculated
DPR: 12  # Will not be recalculated
```

## Alphabetical Reference

---

## **AC** (CALCULATED/USER)
**Usage**: Armor Class
**Type**: Number
**Calculated From**: TIER, ORGANIZATION, THREATS.AC, and ADJUSTMENTS.AC
**Examples**:
```yaml
AC: 13
```

---

## **ACTIONS** (USER)
**Usage**: List of actions the monster can take
**Type**: List of objects with `name`, `description`, and optional `bonus`, `long`, `abbrev`
**Related**: `MULTIATTACK`, `BONUS ACTIONS`, `REACTIONS`
**Examples**:
```yaml
ACTIONS:
  - name: Bite
    bonus: +4
    description: MW A +4, 1d4 pierce.
    long: |
      _Melee Weapon Attack:_ +4 to hit, reach 5 ft., one target.
      _Hit:_ 1d4 piercing damage.
  - name: Crimson Feast
    description: DC 11 CON save or 2d4 necrotic, poisoned.
```

**Description Fields**:
- `description`: Short form for DM reference (required)
- `long`: Full statblock text (optional)
- `abbrev`: Abbreviated form (optional)

---

## **ADJUSTMENTS** (CALCULATED/USER)
**Usage**: Numeric adjustments to base stats
**Type**: Table with stat keys
**Keys**: `AC`, `MAX HP`, `DPR`, `SAVE DC`, `ATTACK BONUS`
**Examples**:
```yaml
ADJUSTMENTS:
  MAX HP: -3
  AC: 1
```

---

## **ALIGNMENT** (USER)
**Usage**: Creature's moral and ethical alignment
**Type**: String
**Examples**:
```yaml
ALIGNMENT: chaotic neutral
ALIGNMENT: typically neutral evil
ALIGNMENT: unaligned
```

---

## **ARMOR** (USER)
**Usage**: Description of armor type
**Type**: String
**Examples**:
```yaml
ARMOR: leather armor
ARMOR: natural armor
ARMOR: false  # For creatures with no armor
```

---

## **ATTACK BONUS** (CALCULATED/USER)
**Usage**: Highest attack bonus among all attacks
**Type**: Number
**Calculated From**: Maximum bonus value from all ACTIONS
**Examples**:
```yaml
ATTACK BONUS: 4
```

---

## **BLIND** (USER)
**Usage**: Indicates the monster cannot see
**Type**: Boolean
**Examples**:
```yaml
BLIND: true
```

---

## **BONUS ACTIONS** (USER)
**Usage**: List of bonus actions available to the monster
**Type**: List of objects with `name` and `description`
**Examples**:
```yaml

  - name: Hide
    description: Dexterity (Stealth) check, +6.
  - name: Disengage
    description: Move w/o provoking attacks.
```

---

## **CHA** (USER)
**Usage**: Charisma ability score
**Type**: Number
**Examples**:
```yaml
CHA: 11
```

---

## **CON** (USER)
**Usage**: Constitution ability score
**Type**: Number
**Examples**:
```yaml
CON: 14
```

---

## **CONDITION IMMUNITIES** (USER)
**Usage**: List of conditions to which the monster is immune
**Type**: List of strings
**Examples**:
```yaml
CONDITION IMMUNITIES:
  - blinded
  - charmed
  - frightened
  - paralyzed
  - poisoned
```

---

## **CR** (CALCULATED)
**Usage**: Challenge Rating
**Type**: String (e.g., "1/4", "2", "1/2")
**Calculated From**: Defensive CR (from MAX HP and AC) and Offensive CR (from DPR, ATTACK BONUS, SAVE DC)
**Examples**:
```yaml
CR: "1/4"
CR: "2"
```

---

## **DEX** (USER)
**Usage**: Dexterity ability score
**Type**: Number
**Examples**:
```yaml
DEX: 15
```

---

## **DPR** (CALCULATED/USER)
**Usage**: Damage Per Round
**Type**: Number
**Calculated From**: TIER, ORGANIZATION, and THREATS.DPR if not provided
**Examples**:
```yaml
DPR: 8
```

---

## **HIT DICE** (USER)
**Usage**: Hit die size
**Type**: String (format: "dN" where N is die size)
**Calculated From**: SIZE (tiny=d4, small=d6, medium=d8, large=d10, huge=d12, gargantuan=d20)
**Examples**:
```yaml
HIT DICE: d8
HIT DICE: d6
```

**Usage**: List of damage types to which the monster is immune
**Type**: List of strings

```yaml
  - fire
  - poison
```

---

## **INT** (USER)
**Usage**: Intelligence ability score
**Type**: Number
**Examples**:
```yaml
INT: 10
```

---

## **LANGUAGES** (USER)
**Usage**: Languages the monster can speak
**Type**: List of strings
**Examples**:
```yaml
LANGUAGES:
  - Common

  - Thieves' Cant
```

---

## **LEGENDARY ACTIONS** (USER)
**Usage**: List of legendary actions (for legendary creatures)
**Type**: List of objects with `name` and `description`
**Examples**:
```yaml
LEGENDARY ACTIONS:
  - name: Detect
    description: Makes a Wisdom (Perception) check.
```

---

## **MAX HP** (CALCULATED/USER)
**Usage**: Maximum hit points
**Type**: Number
**Calculated From**: TIER, ORGANIZATION, THREATS["MAX HP"], SIZE, CON, and ADJUSTMENTS["MAX HP"]
**Examples**:
```yaml
MAX HP: 38
```

---

## **MONSTER TYPE** (USER)
**Usage**: Type classification
**Type**: String
**Common Types**: Aberration, Beast, Celestial, Construct, Dragon, Elemental, Fey, Fiend, Giant, Humanoid, Monstrosity, Ooze, Plant, Undead
**Examples**:
```yaml
MONSTER TYPE: Humanoid (goblinoid)
MONSTER TYPE: Beast
MONSTER TYPE: Ooze
```

---

## **MORALE** (USER)
**Usage**: Morale check information
**Type**: Table with THRESHOLD, TRIGGER, TYPE, DC, and HP (calculated)
**Keys**:
- `THRESHOLD`: Number (0-1) representing HP fraction when morale check triggers
- `TRIGGER`: String describing when check occurs (e.g., "staggered", "bloodied", "critical")
- `TYPE`: String describing behavior on failed check (e.g., "retreat", "rout", "panic")
- `DC`: Morale check difficulty
- `HP`: (CALCULATED) Hit point value at which morale check occurs

**Examples**:
```yaml
MORALE:
  THRESHOLD: 0.5
  TRIGGER: staggered
  TYPE: retreat
  DC: 12
  HP: 7  # Calculated by angrycalc
```

---

## **MULTIATTACK** (USER)
**Usage**: Description of multiattack ability
**Type**: List with one object containing `name` and `description`
**Examples**:
```yaml
MULTIATTACK:
  - name: Multiattack
    description: Two *Shortsword* attacks per round or one *Whirlwind*.
```

---

## **NAME** (USER)
**Usage**: Monster's name
**Type**: String
**Examples**:
```yaml
NAME: Goblin Skirmisher
NAME: Blood Ooze
```

---

## **NOTES** (OPTIONAL)
**Usage**: Freeform notes, typically source citations
**Type**: String (may use YAML multiline format)
**Examples**:
```yaml
NOTES: >-
  FM 309. Modified.
```

**Note**: Consider using `SOURCES` instead for source citations.

---

## **ORGANIZATION** (USER)
**Usage**: Typical encounter grouping
**Type**: String
**Valid Values**: solo, pair, group, gang, mob, party
**Examples**:
```yaml
ORGANIZATION: group
ORGANIZATION: pair
```

---

## **OVERRIDES** (USER)
**Usage**: List of field names that should not be recalculated by `angrycalc`
**Type**: List of strings
**Examples**:
```yaml
OVERRIDES:
  - AC
  - DPR
  - MAX HP
```

---

## **PASSIVE PERCEPTION** (USER)
**Usage**: Passive Perception score
**Type**: Number
**Examples**:
```yaml
PASSIVE PERCEPTION: 9
PASSIVE PERCEPTION: 26
```

---

## **REACTIONS** (USER)
**Usage**: List of reactions available to the monster
**Type**: List of objects with `name` and `description`
**Examples**:
```yaml
REACTIONS:
  - name: Parry
    description: Add 2 to AC against one melee attack that would hit. Must see attacker and wield melee weapon.
```

---

## **RESISTANCES** (USER)
**Usage**: List of damage types the monster resists
**Type**: List of strings
**Examples**:
```yaml
RESISTANCES:
  - cold
  - fire
```

---

## **ROLE** (OPTIONAL)
**Usage**: Monster's tactical role (from MCDM Flee, Mortals!)
**Type**: String
**Examples**:
```yaml
ROLE: ambusher
ROLE: controller
ROLE: defender
```

---

## **SAVE DC** (CALCULATED/USER)
**Usage**: Save DC for monster's abilities
**Type**: Number
**Calculated From**: Maximum savedc value from all ACTIONS, or from TIER, ORGANIZATION, and THREATS["DC"]
**Examples**:
```yaml
SAVE DC: 12
```

---

## **SAVE PROFICIENCIES** (USER)
**Usage**: List of saving throws in which the monster is proficient
**Type**: List of strings (ability names in lowercase)
**Examples**:
```yaml
SAVE PROFICIENCIES:
  - dex
  - wis
```

---

## **SENSES** (USER)
**Usage**: Special senses the monster possesses
**Type**: List of strings
**Format**: "Sense_name distance"
**Examples**:
```yaml
SENSES:
  - Darkvision 60 ft
  - Blindsight 30 ft
  - Tremorsense 60 ft
  - Telepathy 120 ft
```

---

## **SIZE** (USER)
**Usage**: Physical size category
**Type**: String
**Valid Values**: tiny, small, medium, large, huge, gargantuan
**Examples**:
```yaml
SIZE: small
SIZE: medium
```

---

## **SKILLS** (USER)
**Usage**: Skill proficiencies with calculated bonuses
**Type**: List of strings
**Format**: "Skill_name +bonus"
**Bonus Calculation**: ability_modifier + proficiency_bonus × expertise_multiplier
- Proficiency bonus determined by TIER
- Expertise multiplier is 2 if marked with expertise, 1 otherwise

**Examples**:
```yaml
SKILLS:
  - Stealth +6
  - Perception +3
```

---

## **SPEED** (USER)
**Usage**: Narrative speed description
**Type**: String
**Examples**:
```yaml
SPEED: 30 ft.
SPEED: 0 ft., fly 50 ft. (hover)
```

---

## **SPEEDS** (USER)
**Usage**: Numeric movement speeds by type
**Type**: Table with mode keys
**Keys**: WALK, BURROW, CLIMB, FLY, SWIM, HOVER
**Examples**:
```yaml
SPEEDS:
  WALK: 30

SPEEDS:
  WALK: 0
  FLY: 50
  HOVER: true
```

---

## **SOURCES** (USER)
**Usage**: List of source books where this monster appears
**Type**: List of strings (source abbreviations)
**Common Sources**: MM (Monster Manual), VGtM (Volo's Guide), MTF (Mordenkainen's Tome of Foes), etc.
**Examples**:
```yaml
SOURCES:
  - MM
  - VGtM
SOURCES:
  - MM
  - LMoP
  - PotA
```

---

## **STR** (USER)
**Usage**: Strength ability score
**Type**: Number
**Examples**:
```yaml
STR: 12
```

---

## **TEXT** (CALCULATED)
**Usage**: Formatted text renderings for statblock
**Type**: Table with rendering keys
**Keys**: AC, HP, SPEED, CR, "SAVING THROWS", SENSES
**Calculated By**: `angrycalc`
**Examples**:
```yaml
TEXT:
  AC: 13 (leather armor)
  HP: 14 (4d6)
  SPEED: 30 ft.
  CR: 1/4 (50 XP)
  SAVING THROWS: DEX +6
  SENSES: Darkvision 60 ft, Telepathy 120 ft
```

---

## **THREAT** (CALCULATED/USER)
**Usage**: Total threat level
**Type**: Number
**Calculated From**: Sum of all THREATS values
**Examples**:
```yaml
THREAT: 0
THREAT: -1
```

---

## **THREATS** (CALCULATED/USER)
**Usage**: Threat adjustments by stat category
**Type**: Table with stat keys
**Keys**: AC, "MAX HP", DPR, "SAVE DC", "ATTACK BONUS"
**Values**: Threat modifier (-1 for poor, 0 for average, +1 for good, etc.)
**Examples**:
```yaml
THREATS:
  AC: 1
  MAX HP: -1
  DPR: 0
  DC: 0
  ATTACK BONUS: 0
```

---

## **TIER** (USER)
**Usage**: Power level tier
**Type**: String
**Valid Values**: apprentice, journeyman, adventurer, veteran, champion, epic, legendary
**Proficiency Bonuses**: apprentice/journeyman=+2, adventurer=+3, veteran=+4, champion/epic=+5, legendary=+6
**Examples**:
```yaml
TIER: apprentice
TIER: veteran
```

---

## **TRAITS** (USER)
**Usage**: Special abilities and features
**Type**: List of objects with `name`, `description`, and optional `long`, `abbrev`
**Examples**:
```yaml
TRAITS:
  - name: Pack Tactics
    description: Adv with active ally within 5 ft of target.
    long: |
      The rat has advantage on an attack roll against a creature
      if at least one of the rat's allies is within 5 ft. of the
      creature and the ally isn't incapacitated.
  - name: Keen Smell
    description: Adv on Perception (smell).
```

---

## **VILLAIN ACTIONS** (USER)
**Usage**: Villain actions (for villain-tier creatures)
**Type**: List of objects with `name` and `description`
**Examples**:
```yaml
VILLAIN ACTIONS:
  - name: Command Minions
    description: Up to 3 allies within 60 ft can use reactions to move or attack.
```

---

## **VARIANT** (USER)
**Usage**: Variant rules or alternative abilities for the monster
**Type**: List of objects with `name` and `description`
**Examples**:
```yaml
VARIANT:
  - name: Customizing Dragons
    description: You can customize any dragon's stat block to reflect the dragon's unique character.
  - name: Spellcasting
    description: The creature is a 5th-level spellcaster. Its spellcasting ability is Intelligence.
```

---

## **VULNERABILITIES** (USER)
**Usage**: Damage types to which the monster is vulnerable
**Type**: List of strings
**Examples**:
```yaml
VULNERABILITIES:
  - fire
  - radiant
```

---

## **WIS** (USER)
**Usage**: Wisdom ability score
**Type**: Number
**Examples**:
```yaml
WIS: 12
```

---

## **XP** (CALCULATED)
**Usage**: Experience points awarded for defeating the monster
**Type**: Number
**Calculated From**: CR value
**Examples**:
```yaml
XP: 50
XP: 100
```

## Example Monster

Here's a complete example monster YAML:

```yaml
---
NAME: Goblin Skirmisher
TIER: apprentice
ORGANIZATION: group
SIZE: small
MONSTER TYPE: Humanoid (goblinoid)
ALIGNMENT: neutral evil

STR: 8
DEX: 14
CON: 10
INT: 10
WIS: 8
CHA: 8

MAX HP: 14
HIT DICE: d6
ARMOR: leather armor

SPEED: 30 ft.
SPEEDS:
  WALK: 30

SENSES:
  - Darkvision 60 ft

LANGUAGES:
  - Common
  - Goblin

SKILLS:
  - Stealth +6

DPR: 5
THREAT: 0
THREATS:
  AC: 0
  MAX HP: 0
  DPR: 0
  DC: 0
  ATTACK BONUS: 0

MORALE:
  THRESHOLD: 0.5
  TRIGGER: staggered
  TYPE: retreat
  DC: 12

TRAITS:
  - name: Nimble Escape
    description: Can Disengage or Hide as bonus action.

ACTIONS:
  - name: Spear
    bonus: '+4'
    description: _Melee Weapon Attack:_ +4 to hit, reach 5 ft., 1d8-1 piercing.
```
